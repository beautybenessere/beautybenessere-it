# REGOLE BEAUTYBENESSERE.IT

## Regola base progetto
BeautyBenessere.it è un portale/directory nazionale dedicato alle attività Beauty e Benessere.
Il progetto deve mantenere una struttura ordinata, scalabile e documentata, seguendo una logica simile a quella usata per PrenotaQuiOra.

## Regola leggibilità UI
Il progetto non deve usare caratteri piccolissimi.

Linee guida obbligatorie:
- Mai testi sotto i 14px, salvo micro-dettagli davvero secondari.
- Testi principali almeno 16px.
- Descrizioni importanti 16–18px.
- Bottoni, input, filtri e campi ricerca sempre comodi e leggibili.
- Titoli delle card ben visibili.
- Badge leggibili, non microscopici.
- Tabelle e liste admin non devono essere compresse.
- Design premium sì, ma sempre leggibile.
- Elegante sì, ma non piccolo.
- Pulito sì, ma non difficile da leggere.

## Regola stile front-end pubblico
Il front-end pubblico deve avere uno stile:
- elegante;
- premium;
- luminoso;
- caldo;
- rassicurante;
- moderno;
- facile da usare.

Palette indicativa:
- sfondo avorio / bianco caldo;
- card bianche;
- testo antracite;
- accenti oro soft;
- supporto verde salvia;
- bordi chiari e ombre leggere.

Evitare:
- rosa eccessivo;
- stile troppo femminile;
- icone cartoon;
- testi piccoli;
- layout troppo compresso;
- effetto directory vecchia o stile Pagine Gialle.

## Regola stile admin e gestore
Lato admin e dashboard gestore devono ispirarsi allo stile attuale di PrenotaQuiOra:
- blu notte;
- oro;
- card pulite;
- look premium B2B;
- interfaccia ordinata e leggibile.

Il pubblico e il gestionale devono avere identità coerenti ma distinte:
- pubblico: chiaro, caldo, accogliente;
- admin/gestore: blu notte, oro, operativo, professionale.

## Regola struttura ZIP
Ogni ZIP di lavoro deve contenere:
1. file progetto aggiornati;
2. documentazione `.md` aggiornata;
3. stato lavori aggiornato;
4. eventuale report modifiche.

## Regola interventi
Quando si interviene sul progetto:
- modificare solo i file richiesti;
- non toccare sezioni non richieste;
- mantenere coerenza grafica;
- aggiornare la documentazione quando cambiano regole o decisioni importanti;
- mantenere lo ZIP completo e ordinato.
